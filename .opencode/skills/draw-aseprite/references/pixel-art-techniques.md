## 1. 调色板设计

### Hue Shifting（色相偏移）

像素艺术立体感的核心。**不要只靠变亮/变暗同一色相**——阴影偏冷、高光偏暖（反之亦可），整条渐变带色相位移。

- 用 `generate_color_ramp(base_color, hue_shift_degrees=20, lightness_range=0.5)`：默认暗端往一个方向偏、亮端往反方向偏，共 20° 色相跨度。
- 实例：皮肤基底 `#D08560` → 阴影偏红紫 `#7A3A45`，高光偏黄 `#F0C898`，比单调变暗生动。
- 金属：阴影偏蓝灰，高光接近白；布料：阴影偏深色相，高光偏淡。

### 色数控制

| 精灵尺寸 | 建议色数 | 说明 |
|---------|---------|------|
| 16×16 | ≤ 8-12 | 极简，每种材质 3-4 阶 |
| 32×32 | ≤ 16-24 | 每材质 4-5 阶（暗/中暗/中/中亮/亮）|
| 64×64 | ≤ 32-48 | 可加次级高光、环境反射色 |

每条 ramp 3-5 阶（暗/中/亮）是性价比最高的配置。

### 对比度

主体与背景、阴影与高光之间要有足够对比。用 `get_color_stats` 检查实际使用的色数；近重复色（RGB 差 < 20）合并掉。

## 2. 形状构建技法

### 由大到小（Silhouette First）

1. 先用 `fill_area_at` 填一个主色矩形作为剪影占位。
2. 用 `draw_ellipse_at`（头/关节球）、`draw_rectangle_at`（躯干/四肢块）搭出比例骨架。
3. 在骨架上用 `draw_pixels_at` 逐像素修出精确轮廓。
4. 最后点内部细节。

### 像素圆（避免方块感）

小半径圆（r ≤ 4）用 `draw_circle_at` 可能失真，手绘更准。标准像素圆弧参考（1/4 圆，半径递增）：

```
半径3:  ▓▓▓
        ▓   ▓
        ▓   ▓
         ▓▓▓
```

关键：弧的每段斜边长度单调递减，避免 2px 以上的水平/垂直段紧邻。

### 对称部件

画一半 → `flip_layer`(horizontal) 镜像。注意镜像后拼接处可能有线缝，用 `draw_pixels_at` 补中轴线。

## 3. 轮廓与抗锯齿法则

### 禁止抗锯齿

像素艺术**严禁**平滑边缘。所有轮廓必须是硬边像素。`draw_*` 工具默认就是硬边，不要用任何模糊滤镜做轮廓。

### Jaggies（无意锯齿）识别与修复

Jaggies = 轮廓上不该有的阶梯感。常见形态：

```
坏（2px 直段紧邻对角）:     好（斜边渐变）:
  ▓▓                          ▓
  ▓▓                          ▓▓
  ▓                           ▓
  ▓                             ▓
```

规则：**任何斜边上，单步水平/垂直段不超过 2px，且相邻段长度差 ≤ 1。** 自检时 `export_frame` 8× 放大看轮廓。

### 描边（Outline）决策

- **外轮廓**：`outline_cel`（1px 黑/深色）让精灵在任何背景上可读，角色/UI 图标推荐。
- **选择性描边**：只在朝光源反方向描边（暗侧），亮侧不描——用 `outline_native`(place=outside) 后手动擦亮侧。
- **无轮廓**：极简/复古风（如 GameBoy 风）可省轮廓，靠色块对比区分。

### 内轮廓（色块分界）

部件之间（如衣服与皮肤）用 1px 深色线或阴影色分界，增强层次。

## 4. 抖动（Dithering）应用

用两种色的规则点阵模拟第三色/渐变，不引入新色（色板纪律友好）。

| 工具 | 用途 |
|------|------|
| `apply_dither_gradient` | 两种色的方向渐变（天空/金属表面/球体明暗）|
| `apply_dither_pattern` | 均匀混合（density 控制比例；石头纹理、草地、皮肤阴影过渡）|

- **density 0.5** = 棋盘格（视觉中间色）。
- 渐变方向：球体/圆柱用径向（手动分段做水平抖动带），天空/地平线用垂直。
- 32×32 以下慎用抖动（像素太粗显脏）；64×64 以上效果好。

## 5. 光影原则

### 单一光源

定一个主光源方向（常用左上或正上），全程一致。阴影在背光侧，高光在向光侧。

### 明暗五调（简化为像素可用）

像素精灵通常用 3-5 阶：

1. **核心阴影**（Core Shadow）：最暗，背光侧边缘。
2. **中间调**（Mid Tone）：基底色。
3. **高光**（Highlight）：向光侧，面积小。
4. **反射光**（Bounce Light，可选）：阴影边缘内侧一线亮色，增加体积感。
5. **环境光**（Ambient，可选）：朝向天空的面偏冷。

### 落地阴影

角色脚下用半透明椭圆（`draw_ellipse_at` 低透明 + `set_layer_blend_mode` multiply）增强接地感，单独一层方便调整。

## 6. 动画原则

### 帧率与时长

- 待机（idle）：4-6 帧，每帧 150-200ms，`pingpong` 来回。
- 走路（walk）：6-8 帧，每帧 80-120ms。
- 攻击（attack）：4-6 帧，每帧 60-100ms，关键动作帧（接触帧）可缩短。
- `set_tag` 的 direction：循环用 forward/pingpong，单次用 forward。

### 像素动画特有原则

- **保持剪影可读**：每帧剪影都要能认出是什么动作（剪影测试）。
- **体积保持**：角色体积跨帧一致，不要忽大忽小（除刻意 squash/stretch）。
- **次级动作**：主体动，附属物（头发/披风/武器）滞后跟随，用 `tween_cel_positions_eased` 加缓动。
- **接触帧停顿**：攻击/落地的接触帧时长略长（set_frame_duration），增强打击感。

### 验证

- `render_onion_skin`：当前帧叠前后帧残影，看运动轨迹是否流畅、无抖动。
- `compare_frames`：相邻帧差异应在合理范围——0 差异说明漏画，差异过大说明跳变。

## 7. 常见资产类型模板

### 角色立绘（32×32）

层序：Outline / Effects / Equipment / Clothing / Body / Background。
- 身体剪影（头身比 1:2 或 1:3，Q版可 1:1.5）。
- 每材质一条 ramp（皮肤/头发/主服/辅服/金属）。
- 左上光源，核心阴影在右下。
- `outline_cel` 描边。

### 物品图标（16×16 / 32×32）

- 单一主色 + 1 阴影 + 1 高光，色数极简。
- 左上 45° 光源，右下阴影。
- `outline_cel` 描边保证在 UI 槽位可读。
- 若需统一风格集，先 `apply_palette_preset` 限制全局色。

### UI 元素

- 9-patch：用 `create_slice` + `set_slice_center` 标记可拉伸区。
- 按钮：底色 + 1px 内高光（亮色）+ 1px 外阴影（暗色）做立体感。

### 场景瓦片（Tilemap）

- `create_tilemap_layer` 建瓦片层。
- `draw_on_tile` 画瓦片素材（草地/石头/水），tile_index 自动递增。
- `set_tiles` 摆放瓦片。
- 瓦片边缘做无缝拼接（避免可见接缝）：边缘像素与相邻瓦片连续。
