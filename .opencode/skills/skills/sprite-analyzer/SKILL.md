# Sprite Analyzer

逐张分析精灵图，生成结构化 Markdown 文档。

## 工作流

```
1. 扫描目录 → 收集所有图片路径和元数据
2. 逐张分析 → 每张图独立完成全部分析后写入文档
3. 输出文档 → 写入指定的 .md 文件
```

### 步骤 1：收集图片元数据

用 `sips` 批量获取尺寸和透明通道信息：

```bash
for f in <sprites_dir>/*.png; do
  echo "=== $(basename $f) ==="
  sips -g pixelWidth -g pixelHeight -g hasAlpha "$f"
done
```

同时扫描子目录（如 `ui/`）。

### 步骤 2：逐张分析（核心循环）

**严格约束：每次只分析一张图，写完再分析下一张。**

对每张图执行：

1. **获取元数据** — `sips -g pixelWidth -g pixelHeight -g hasAlpha <file>`
2. **AI 视觉分析** — 调用 `zai-mcp-server_analyze_image`，prompt 要点：
   - 图片内容（角色/道具/地形/UI）
   - 精灵表布局（行数×列数，帧尺寸）
   - 动画帧分布（每行动作、帧数）
   - 像素艺术细节（颜色、阴影、高光、透明通道）
   - 是否有旋转/变形动画效果
3. **写入文档** — 立即将分析结果追加到输出文件

### 步骤 3：输出格式

每个精灵条目包含以下结构：

```markdown
### <filename> — <中文名称>

| 属性 | 值 |
|------|-----|
| 尺寸 | <宽> × <高> px |
| 布局 | <行> × <列>（<帧尺寸> 网格） |
| 帧数 | <总数> 帧 |
| 透明通道 | 有/无 (RGBA) |
| 主色调 | <颜色描述> |

**动画行分布：**（精灵表适用）

| 行号 | 帧数 | 动画类型 | 说明 |
|------|------|----------|------|

**内容描述：** <文字描述图片内容>

**Godot 使用方式：**
- 导入方式
- 切分参数（hframes/vframes）
- 代码示例（如适用）

**适用场景：**
- 场景1
- 场景2
```

## 分析 Prompt 模板

调用 `zai-mcp-server_analyze_image` 时使用以下 prompt 结构：

```
这是一张游戏精灵图资源，尺寸<W>x<H>像素，带/不带透明通道。
请详细分析：
1. 精灵表布局（几行几列，每帧多大）
2. 包含哪些动画动作/元素变体
3. 每行对应什么动作，几帧
4. 颜色、阴影、高光等像素艺术细节
5. 是否有空白帧或特殊帧
6. 在2D游戏中的用途
```

## 注意事项

- **单张串行**：必须等一张图的分析结果写入文档后，才能开始下一张
- **去重**：如果输出文件已有相同图片的条目，合并为更详细的版本，不重复添加
- **文档目录**：输出文件顶部包含带锚点链接的目录
- **元数据来源**：`sips` 提供精确尺寸，AI 分析提供内容识别，二者结合
- **引用工具**：Read 工具无法直接显示图片，必须用 `zai-mcp-server_analyze_image` 分析
