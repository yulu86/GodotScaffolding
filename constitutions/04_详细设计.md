# 📐 详细设计宪法

> 详细设计阶段：基于架构设计，进行 UI 开发和场景开发的详细设计。
> 本阶段产出开发指导文档，并通过人机协作完成场景构建。

## 适用 Skill

| Skill | 职责 | 可修改文件 |
|-------|------|-----------|
| `godot-scene` | 场景创建与修改指导 | 无（输出操作指导，由用户执行） |

## 规则

### UI 开发

- **P1-15** UI 开发流程（须基于 pencil 设计稿）：
  ```
  pencil_batch_get (读取设计稿)
    → task(category="visual-engineering", load_skills=["godot-scene"]) (场景构建与布局)
    → task(category="deep", load_skills=["godot-developer"]) (脚本实现)
    → godot-mcp_run_project + godot-mcp_get_debug_output (验证)
  ```

### 场景开发

- **P1-16** 场景开发**必须**采用人机协作模式：
  ```
  AI 输出详细操作指导（节点结构、属性配置、信号连接）
    → AI 暂停执行，等待用户操作
    → 用户在 Godot 编辑器中创建/编辑 .tscn
    → 用户告知 AI "继续"
    → AI 验证并继续下一步
  ```
- **P1-16a** AI 禁止通过 MCP 工具直接创建/修改 .tscn 文件，必须输出操作指导由用户执行
- **P1-16b** 每次操作指导必须包含：节点路径、节点类型、属性设置、信号连接方式
- **P1-16c** AI 必须在输出指导后明确暂停，显示等待提示，不可自行继续
