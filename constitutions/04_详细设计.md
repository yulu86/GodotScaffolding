> 详细设计阶段：基于架构设计，进行 UI 开发和场景开发的详细设计。
> 本阶段产出开发指导文档，并通过人机协作完成场景构建。

## 规则

### UI 开发

- **P1-15** UI 开发流程（须基于 pencil 设计稿）：
  ```
  pencil_batch_get (读取设计稿)
    → AI 创建场景基本框架（节点树骨架，禁止设置可视化属性）
    → 输出操作指导到 tmp/（图片加载、位置微调、布局调整等可视化操作由用户执行）
    → 用户在 Godot 编辑器中完成可视化操作并确认
    → task(category="deep", load_skills=["godot-best-practices"]) (脚本实现)
    → godot-mcp_run_project + godot-mcp_get_debug_output (验证)
  ```

### 场景开发

- **P1-16** 场景开发**必须**采用 **AI 创建框架 + 用户可视化操作** 的人机协作模式：
  ```
  AI 通过 MCP 创建场景基本框架（节点树骨架、基本节点类型）
    → AI 输出详细操作指导到 tmp/ 目录
    → AI 继续执行 Story 中的其他任务（脚本编写、测试等），不暂停
    → Story 中所有任务执行完成后，AI 暂停执行，等待用户编辑场景
    → 用户在 Godot 编辑器中执行可视化操作（图片加载、位置微调、布局调整等）
    → 用户告知 AI "继续" / "完成"
    → AI 检视场景（节点树结构、信号连接等）并执行测试用例验证
  ```
- **P1-16a** AI 禁止通过 MCP 工具执行以下需要可视化确认的操作：
  - 设置精确位置、锚点、偏移等布局属性
  - 加载图片/纹理等资源
  - 配置动画关键帧
  - 微调 Container 间距、边距等参数
- **P1-16b** 每次操作指导必须包含：节点路径、节点类型、属性设置、信号连接方式、操作意图说明
- **P1-16c** 场景创建/编辑**不需要**暂停其他并行任务。AI 创建场景框架后，可继续执行 Story 中的脚本编写、测试等其他任务
- **P1-16d** 当 Story 中的所有任务（场景框架创建、脚本编写、测试等）都执行完成后，AI **必须暂停执行**，等待用户在编辑器中编辑场景
- **P1-16e** 用户显式确认场景编辑完成后，AI **必须**执行以下步骤：
  1. 检视场景（检查节点树结构、信号连接、脚本挂载等是否正确）
  2. 运行相关测试用例验证功能正确性
  3. 确认无误后方可继续下一个 Story
- **P1-16f** 场景开发任务禁止在子agent 中执行，必须由主会话直接操作
