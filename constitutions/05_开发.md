# 💻 开发宪法

> 开发阶段：基于详细设计，执行 TDD 实现、编码和重构。
> 本阶段通过 TDD 微循环交付 .gd 脚本文件。

## 适用 Skill

| Skill | 职责 | 可修改文件 |
|-------|------|-----------|
| `godot-developer` | TDD 代码实现 | `.gd` 脚本文件 |

## 前置条件

- **P1-18** 开发阶段开始前，**必须**读取以下上游文档，确保实现基于已确认的需求、架构和详细设计：
  - `docs/01_gdd/` 下的需求规格文档（游戏设计文档、功能需求文档）
  - `docs/02_analysis/` 下的需求分析文档（技术可行性分析、性能需求分析）
  - `docs/03_arch/` 下的架构设计文档（架构概要设计、模块设计、状态机设计）
  - `docs/05_guide/` 下的详细设计文档（功能开发指导）
- **P1-18a** 若上述文档不存在，**禁止**进入开发阶段，必须提醒用户先完成上游阶段
- **P1-18b** 代码实现必须与架构设计文档中定义的模块划分、接口规范保持一致

## 规则

### Story 开发流程

- **P1-14** **必须以 Story 粒度进行开发**，每次仅开发 1 个 Story，禁止并行开发多个 Story
- **P1-15** 每个 Story 开发前，**必须**输出以下内容：
  1. Story 的完整内容（需求描述、验收标准）
  2. AI 对该 Story 的理解和实现思路（包括涉及哪些文件、预期变更范围、技术方案）
  - 输出后暂停，等待用户确认思路后方可开始编码
- **P1-16** 代码开发**必须**使用 TDD 微循环（见下方 TDD 微循环章节），遵循 Red → Green → Refactor 流程
- **P1-17** 代码开发完成后，**必须**进行代码检视（Code Review），检查项包括：
  - 是否符合编码规范（P1-3 ~ P1-5）
  - 是否存在未处理的边界条件
  - 是否有性能隐患
  - 命名是否清晰、逻辑是否易读
- **P1-18** 每个 Story 开发完成后，**必须**检查是否生成了必需的 `.uid` 文件：
  - 新建的 `.gd` 脚本文件必须有对应的 `.uid` 文件
  - 若发现缺失，必须通过 Godot 编辑器重新扫描生成
- **P1-19** 每个 Story 完成后，**必须暂停执行**，向用户汇报完成情况，等待用户确认后再继续下一个 Story

### TDD 微循环

- **P1-1** 使用 `godot-developer` 技能 + TDD 微循环：
  ```
  godot-test (编写测试)
    → godot-developer (实现代码)
    → minimal-godot_get_diagnostics (语法检查)
    → $GODOT_HOME -s addons/gut/gut_cmdln.gd -gexit (运行测试)
    → godot-developer (重构优化)
  ```

- **P1-2** 使用 `context7` 工具查询 API 文档：`context7_resolve-library-id` → `context7_query-docs`

### 编码规范

- **P1-3** 优先 AnimationPlayer 节点
- **P1-4** Singleton .gd 文件禁止 `class_name`
- **P1-5** 禁止三元运算符，使用 `if...else`

### 场景文件约束

- **P1-6** **禁止** AI 创建或修改 `.tscn` 场景文件（包括通过 MCP 工具 `godot-mcp_create_scene`、`godot-mcp_add_node`、`godot-mcp_save_scene` 等）
- **P1-7** 当任务需要创建或修改场景时，AI 必须：
  1. 输出详细的场景操作指导（包括节点树结构、节点类型、属性设置、信号连接等）
  2. **暂停执行**，等待用户手动完成场景操作
  3. 用户确认完成后，方可继续后续任务

### 代码重构

- **P1-11** 重构前分析：
  ```
  lsp_symbols (理解文件结构)
    → lsp_find_references (查找所有引用)
    → ast_grep_search (搜索代码模式)
    → ast_grep_replace --dryRun (预览替换)
  ```

### 性能优化

- **P1-13** 性能优化流程：
  ```
  minimal-godot_get_diagnostics (排除语法问题)
    → minimal-godot_scan_workspace_diagnostics (全项目扫描)
    → ast_grep_search (定位性能模式，如 _process 中的对象创建)
    → godot-mcp_run_project + godot-mcp_get_debug_output (运行时分析)
  ```
