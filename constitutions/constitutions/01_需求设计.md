# 🎨 需求设计宪法

> 游戏设计阶段：资源分析 → 游戏设计文档 → 功能需求 → UI 界面设计。
> 本阶段产出 GDD、功能需求文档和 .pen 界面设计稿。

## 适用 Skill

| Skill | 职责 | 可修改文件 |
|-------|------|-----------|
| `sprite-analyzer` | 精灵图资源分析 | 分析输出文档（.md） |
| `godot-ui-design` | 界面设计（Pencil） | `docs/07_design/*.pen` |

## 规则

### 资源分析

- 涉及精灵图资源时，**必须**先由 `sprite-analyzer` 分析资源结构和动画帧信息

### 游戏设计

- **P1-14** 游戏设计阶段，UI 界面**必须**使用 `pencil` 工具设计：
  ```
  task(category="deep", load_skills=["godot-ui-design"]) (Pencil 界面设计)
  ```
- **P1-14a** 界面设计稿存放在 `docs/07_design/` 目录，格式为 `.pen` 文件
- **P1-14b** 未经 pencil 设计的界面禁止进入开发阶段

## 交付件

| 交付件 | 命名格式 | 存档路径 |
|--------|---------|---------|
| 游戏设计文档 | `01_游戏设计文档.md` | `docs/01_gdd/` |
| 功能需求文档 | `{序号}_功能需求_{功能名}.md` | `docs/01_gdd/` |
| 界面设计稿 | `{序号}_{界面名}.pen` | `docs/07_design/` |

## 工作流

```
sprite-analyzer (分析精灵图资源，提取元数据和动画帧信息)
  → godot-ui-design (使用 Pencil 设计 UI 界面，输出 .pen 设计稿)
  → 输出 GDD 和功能需求文档
```
